Configuring Git
git config --global user.name "My Name"
git config --global user.email "someone@email.com"
git config --list

Clone & Status
	Clone - Cloning a repository on our local machine
		Git Command -> git clone <link>
	Status - display the state of the code
		Git Command -> git status
		1) Untracked -> new Files that git doesn't yet track
		2) Modified -> changed
		3) Staged -> File is ready to be committed
		4) Unmodified -> unchanged
	

cd command -> change directory

ls command -> It is used for listing all the files in the Folder/Repository
ls -a command -> It is same as ls but also display hidden files and Folder/Repository

Add & Commit
add -> add new or changed files in your working directory to Git staging area.
	command -> 1) git add <file name> 2) git add . (For all files) 
	commit -> it is the record of change
	command -> git commit -m "Some Message"

Push Command
push -> Upload local Repository/Folder content to remote(GitHub) repo
	command -> git push origin main

Init Command
	init -> Used to create new git repo
	Commands -> git init
		    git remote add origin <Link>
		    git remote -v (To Verify remote)
		    git branch (To check branch)
		    git branch -M main (To rename branch)
		    git puch origin main
                    git puch -u origin main (If your pushing in same branch frequently so user can directly use -> git push)

Branch Commands
	-> git branch (to check branch)
	-> git branch -M main (to rename the branch)
	-> git checkout <- branch name -> (to navigate)
	-> git checkout -b <- new branch name -> (to create new branch)
	-> git branch -d <- branch name -> (to delete branch)

Merging Code
	Way-1
		-> git diff <- branch name -> (to compare commit, branches, files & more)
		-> git merge <- branch name -> (to merge 2 branches)
	Way-2
		Create PR

-> git pull origin main
used to fetch and donwload  content from a remote repo and immediately update and local repo to match that content.
